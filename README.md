# 포켓몬 싱글 3선발 도우미

6마리 지참 → 3마리 선발(팀 프리뷰) 방식의 싱글 배틀에서, 내 팀과 상대 팀을 넣으면 타입 커버리지·선공·약점 노출을 계산해 **최적의 3마리 조합**을 추천하는 도구.

> **🌐 웹에서 바로 사용:** https://hchee99.github.io/pokemon-picker/
> **📱 안드로이드 앱(APK) 다운로드:** [Releases 페이지](https://github.com/hchee99/pokemon-picker/releases/latest) — 다운로드 후 설치 시 "출처를 알 수 없는 앱 허용"이 필요합니다

두 가지 형태로 쓸 수 있습니다:

| 구성 | 설명 |
|---|---|
| **웹 계산기** (`index.html`) | 브라우저만 있으면 되는 단일 HTML 파일. 서버·인터넷 불필요, 데이터는 브라우저 안(localStorage)에만 저장 |
| **안드로이드 오버레이 앱** (`android/`) | 게임 중 팀 프리뷰 화면을 캡처해 **상대 6마리를 자동 인식**하고, 게임 위 오버레이로 후보 확인 → 추천까지. 게임을 벗어나지 않아 통신 끊김 없음 |
| **인식 프로토타입** (`prototype/`) | 인식 알고리즘을 PC에서 검증하는 Python(OpenCV) 스크립트들 |

## 웹 계산기 (`index.html`)

### 주요 기능
- **최적 3마리 추천** — 가능한 조합을 전부 계산해 커버리지·선공·약점 기준으로 순위화
- **도구·기술·스피드 버프·특성 설정** — 포켓몬마다 지닌 물건 1개 + 기술 타입 4개 + 순풍·용춤 등 스피드 버프 + 특성(부유 등)을 넣어 실제 커버리지·선공으로 계산 (내 팀·상대 팀 모두 ⚙로 설정, 빌드는 각각 따로 저장)
- **주요 특성 계산 반영** — 부유(땅 무효), 저수·축전·피뢰침·마중물(물/전기 무효), 타오르는불꽃(불꽃 무효), 초식(풀 무효), 두꺼운지방(불꽃·얼음 반감) 등. 상대 카드 상세에는 약점 타입도 함께 표시
- **메가 1마리 제한** — 한 배틀에 메가진화 1마리 규칙을 추천에 자동 적용
- **📌 필수 선발 고정** — 내 팀 슬롯의 📌을 켜면 그 포켓몬을 반드시 포함한 조합만 추천
- **사진/타입 그리드로 고르기** — 이름을 모를 때 각 슬롯 🔍 버튼으로 포켓몬 사진 그리드를 열어 검색·클릭 선택. 사진은 도감 붙여넣기 시 함께 저장되며, 없으면 타입색 타일로 대체
- **팀 프리셋 + 내 팀 자동 저장** — 자주 쓰는 팀을 이름 붙여 저장/불러오기, 마지막 팀·📌 상태는 자동 복원
- **상세 설명** — 추천 카드를 클릭하면 상대별 처리(누가·무슨 타입으로·몇 배·선공/후공)를 표로
- **데이터 관리** — 도감 붙여넣기 / 직접 추가 / JSON 내보내기·가져오기

### 사용법
1. `index.html`을 브라우저로 열기 (더블클릭 — 인터넷 불필요)
2. 처음엔 포켓몬 데이터 채우기
   - **도감 붙여넣기**: 포켓몬 도감 표를 드래그 복사 → 붙여넣기 → 추가 (텍스트·HTML 모두 인식, 중복 자동 제거)
   - **직접 추가**: 목록에 없는 포켓몬은 이름·타입·(스피드) 직접 입력
   - **가져오기**: 기존 JSON이 있으면 붙여넣거나 파일로 불러오기
3. (선택) 내 포켓몬 슬롯의 **⚙** 버튼으로 도구·기술 타입·스피드 버프 설정
4. 내 팀 6칸 + 상대 팀 6칸에 이름 입력 (자동완성)
5. **최적 3마리 조합 찾기** → 추천 카드 확인, 클릭하면 상대별 상세가 펼쳐짐

### 결과 지표
- **커버 X/6** — 상대를 2배↑로 때리는 수 (4배는 별도 표시)
- **선공 커버** — 스피드로 앞서면서 2배↑로 때리는 상대 수 (구애스카프 ×1.5, 순풍·랭크업 등 스피드 버프 배율도 함께 반영)
- **위험** — 상대 자속에 내 포켓몬이 2배↑로 맞는 매칭 수 (낮을수록 안전)

추천 점수 = `커버 × 100 + 선공 커버 × 10 + 4배 × 4 − 위험 × 6` (높은 순 정렬).

### 계산 규칙
- 기술 타입을 설정하면 그 타입들로 커버를, 미설정 시 자속 타입으로 커버를 계산
- 도구는 스피드에 영향을 주는 것(구애스카프 등)만 계산에 반영, 나머지는 표시용
- 스피드 버프(순풍 ×2, 용춤 등 랭크+1 ×1.5, 랭크+2~+6 등)는 배율로 스피드 계산에 반영되며, 도구 효과와 중첩 적용됨
- 선공 판정은 내 실질 스피드와 상대 실질 스피드(상대 스카프·버프 포함)를 비교, 위험(방어) 판정은 상대 기술 타입(미설정 시 상대 자속)을 기준으로 계산
- 메가진화(이름이 `메가`로 시작, 단 `메가니움`은 제외)는 3마리 중 1마리까지만
- 이름이 같고 타입이 다른 지역폼은 `윈디 (불꽃/바위)`처럼 자동 구분 저장

## 안드로이드 오버레이 앱 (`android/`)

온라인 배틀 중 다른 앱으로 전환하면 통신이 끊기는 문제를 피하기 위해, **게임 위에 떠 있는 오버레이**로 동작합니다.

### 흐름
1. 앱에서 **시작 ▶** → 화면 캡처 권한 1회 동의 → ⚡ 버블이 뜸
2. 팀 프리뷰(선출 화면)에서 **⚡ 탭** → 화면을 메모리에서만 캡처 (**파일 저장 없음**, MediaProjection)
3. 상대 6마리의 타입을 자동 인식 → **왼쪽 패널**에 슬롯별 후보 표시 (오른쪽 상대 정보를 가리지 않음)
4. 후보 탭으로 확정. **메가진화가 있는 포켓몬은 보라색 `M` 칩**이 함께 떠서, 탭하면 메가 타입/스피드 기준으로 계산 (예: 갸라도스 물/비행 → 메가갸라도스 물/악)
5. **추천 보기** → 웹 계산기가 오버레이로 열리며 상대가 자동 입력되고 최적 3마리가 계산됨

### 인식 원리 (2단계)
- **1순위 — 심볼 모양 매칭**: 타입 아이콘 속 흰 심볼(불꽃·물방울·번개…)을 20×20 마스크로 떠서 18타입 템플릿과 모양 일치도(IoU) 비교. 아이콘 위치는 화면 비율로 고정. **색이 아닌 형태 기반이라 경기장 조명이 바뀌어도 흔들리지 않음** (데스크톱 검증 54/54)
- **2순위 — 색 분류 폴백**: 심볼이 가려져 매칭이 실패하면(예: 스프라이트 꼬리가 아이콘을 덮음) 픽셀 색 최근접 분류로 추정하고, `⚠ 가림 의심` 표시와 함께 후보를 넓게 제시
- 캡처는 0.7초 간격 **2프레임**을 찍어 병합 — 배경 레이저/애니메이션이 순간적으로 아이콘을 가리는 경우 보완
- 인식 실패·오류는 앱 내 **문제 로그 공유** 버튼으로 로그를 보내 원격 진단 가능 (슬롯별 타입 픽셀 수·주요 색상값 기록)

### 빌드
- 요구: JDK 17, Android SDK (platform 34), Gradle 8.x
- `android/app/src/main/assets/pokedex.json` 필요 — **저장소에 포함되지 않음** (웹 계산기의 "내보내기" JSON을 그대로 사용)
- `android/local.properties`에 `sdk.dir` 지정 후:
  ```
  cd android && gradle assembleDebug
  ```
- 권한: 다른 앱 위에 표시(오버레이), 화면 캡처(MediaProjection), 알림

## 인식 프로토타입 (`prototype/`)

PC에서 인식 알고리즘을 검증/보정하는 Python 스크립트 (요구: `opencv-python`, `numpy`).

| 파일 | 역할 |
|---|---|
| `glyph_match.py` | **심볼 모양 매칭** — 템플릿 생성(`glyph_templates.json`) + 정확도 검증. 앱의 1순위 인식기와 동일 로직 |
| `type_icons.py` | 색 기반 타입 분류 (폴백 로직) + 패널 위치 자동 검출 |
| `match.py`, `rank_test.py` | 스프라이트 색 히스토그램 비교 실험 (후보 순위화) |
| `sample_colors.py`, `dump_region.py` 등 | 색 보정·좌표 측정 도구 |

템플릿을 다시 만들려면 정답을 아는 팀 프리뷰 스크린샷을 `prototype/`에 두고 `glyph_match.py`의 `GROUND_TRUTH`에 정답을 적은 뒤 실행 → `glyph_templates.json`을 `android/app/src/main/assets/`로 복사.

## 데이터 저장
- 웹: 도감·빌드·프리셋·내 팀 모두 브라우저 localStorage에 자동 저장 (같은 브라우저에서 유지)
- **내보내기/가져오기 JSON에 도감 + 빌드 포함** → 다른 기기·브라우저·앱으로 옮길 때 사용
- 앱: 캡처는 메모리에서만 처리되어 **이미지 파일이 저장되지 않음**. 데이터는 어떤 서버로도 전송되지 않고 기기 안에만 있음
