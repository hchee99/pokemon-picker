# 인식 프로토타입 (1단계 검증)

APK를 만들기 전에 **"팀 프리뷰에서 상대 6마리를 스프라이트로 인식할 수 있는가"** 만 PC에서 값싸게 검증하는 도구.

## 실행 순서
1. 파이썬 설치 후:
   ```
   pip install opencv-python numpy
   ```
2. 사이트에서 **"JSON 파일로 저장"** 한 `pokedex.json` 을 이 폴더에 둔다.
3. 팀 프리뷰 **스크린샷**을 이 폴더에 `shot.png` 로 저장.
4. 실행:
   ```
   python match.py shot.png
   ```

## 결과 보는 법
- `debug_crops/slot1~6.png` — 상대 6칸을 어떻게 잘랐는지 (크롭 위치 확인)
- 콘솔 — 칸별 top-3 후보 + 점수(ORB 매칭 수 / 템플릿 상관도)

## 크롭이 어긋나면
`match.py` 상단 `CONFIG` 의 좌표(화면 비율)를 조정 후 재실행.
- `COL_X0, COL_X1` : 스프라이트가 있는 가로 구간
- `ROW_TOP, ROW_BOT` : 6칸이 차지하는 세로 구간

## 판정 기준 (이 단계의 목표)
- top-1 이 대부분 정답이면 → 순수 스프라이트 매칭으로 충분, 안드로이드로 진행
- top-3 안엔 들지만 top-1 이 자주 틀리면 → 타입 아이콘 판독으로 후보를 좁히는 2단계 추가
- top-3 에도 자주 없으면 → 배경 제거/특징매칭 방식 재설계 필요

첫 실행 시 `refs/` 폴더에 레퍼런스 스프라이트가 캐시된다(이후 재사용).
